# Структура проекта
## Application
Приложение (Application) представляет собой c++ класс, который содержит ссылку на структуру SDL_Window, создаваемую библиотекой SDL и представляющую собой контекст рендеринга или окно с поддержкой OpenGL API, используемого классом Renderer данного проекта.
Так же содержит экземпляр registry библиотеки EnTT, который содержит все объекты программы
Основной функционал представляет главный цикл программы
## Resources loader
Загрузчик ресурсов (Resources loader) представлет собой C++ класс, служащий для загрузки json представления ресурсов и поставления из в виде экземпляров классов(объектов) для парсинга объектов он может использовать функционал Renderer и SDL_Image(для записи медиа в видеопамять), Rapidjson(для работы с json), LuaCPP(для создания lua_state файлов для скриптов) и SDL_AudioMixer для загрузки аудиофайлов в оперативную память.
## Renderer 
Рендер движок (Renderer) являюется C++ классом, который оперирует загрузкой и освобождением видеопамяти и всеми обращениями к видеокарте посредством OpenGL функций через SDL, так как содержит ссылку на структуру SDL_Renderer
## Scripts Manager
Менеджер скриптов (Script) C++ класс, который занимается компиляцией и исполнением lua скриптов, загруженных Resources manager'ом главным образом использует LuaCPP для вызовов библиотеки lua
## haste_n_slash.h
Содержит общие для всех систем функции и данные
# Жизненный цикл
В общем виде жизненный цикл прокраммы выглядит так:
1. Начало программы
2. Загрузка данных
3. Создание окна
4. Создание контекста
5. Создание движка рендера
6. Начало главного цикла
7. Обработка ввода
8. Обновление систем
9. Обновление скриптов
10. Обновление рендера
11. Конец главного цикла
12. Уничтожение движка рендера
13. Уничтожение окна
14. Очистка выделенной памяти
15. Конец программы
